iOS Webkit oddities - "-webkit-overflow-scrolling: touch"

When adding -webkit-overflow-scrolling: touch to an element that has overflow: auto and contains an element that has overflow: hidden on it which contains elements that are positioned using translate3d, elements disappear. As shown in this example, this also works when adding the webkit-overflow-scrolling property later. Please note that both the scroll wrapper and the inner container have a fixed with, so this is truly a rendering bug.
